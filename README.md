这份文档必须站在 **AI Agent 的执行视角** 编写，而不是普通开发教程。我的角色只在项目初期深度参与，例如：

1. 说明我想做什么；
2. 补充项目背景；
3. 明确项目目标；
4. 生成并确认需求；
5. 指出需求中的问题并要求修正；
6. 确认最终开发方向。

在需求确认之后，我不再参与具体开发细节，只会持续发送类似 **“下一步”** 的指令。之后的规划、拆解、编码、测试、修复、文档沉淀、阶段验收，都必须由 AI Agent 按照文档自动推进。

## 文档目标

请生成一份可以长期驱动 AI Agent 开发工程级项目的完整 Skill / Prompt 文档。该文档不能依赖单次对话上下文，必须能够在上下文丢失、上下文溢出、切换 Agent、切换工具或重启会话后继续使用。

## 项目开发方式要求

整个项目不能一步到位完成，必须采用**分阶段、分模块、分时间切片**的方式推进。

请将项目拆成以下层级：

```text
项目总目标
  → 阶段路线图
    → 功能模块
      → 子模块
        → 时间切片任务
          → 执行步骤
          → 验收标准
          → 完成标记
          → 问题记录
          → 下一步任务
```

每个功能模块都应按“大型工程模块”处理，不允许过度简化。请使用**保守时间估算 / 最长时间估算**来分配任务，不要使用乐观估算。

---

## 时间切片要求

每个开发流程都必须包含多份时间切片。

每个时间切片必须说明：

1. 时间切片编号；
2. 所属阶段；
3. 所属模块；
4. 当前任务目标；
5. 本次只允许完成的工作范围；
6. 禁止顺手扩展的范围；
7. 输入依赖；
8. 输出产物；
9. 需要修改或新增的文件；
10. 需要执行的命令；
11. 需要通过的测试；
12. 验收标准；
13. 完成判定；
14. 未完成时的处理方式；
15. 下一时间切片的进入条件。

当某个时间切片完成后，必须进行状态标记。标记格式必须严格，例如：

```text
[未开始]
[进行中]
[阻塞]
[待验证]
[已完成]
[需返工]
```

不能只因为代码已经写完就标记为完成。只有当代码能够运行、测试通过、验收标准满足、相关文档更新完成之后，才能标记为 `[已完成]`。

---

## 完成判定规则

AI Agent 必须使用明确的完成标准，不能自我感觉完成。

每个任务必须至少满足以下条件，才能被标记为完成：

1. 代码已实现；
2. 代码能够正常运行；
3. 关键路径测试通过；
4. 没有明显运行时报错；
5. 与需求文档一致；
6. 没有破坏已有功能；
7. 相关文档已更新；
8. 任务状态文件已更新；
9. 下一步任务已明确。

如果其中任何一项不满足，任务必须标记为：

```text
[待验证]
[阻塞]
[需返工]
```

而不是 `[已完成]`。

---

## 防止死亡优化循环

AI Agent 必须避免无限优化、无限重构、无限修复。

请在文档中加入明确规则：

* 每个时间切片只能完成任务清单中定义的工作；
* 不允许临时扩展需求；
* 不允许在没有用户确认的情况下更换技术栈；
* 不允许为了“更优雅”而重写已通过验收的模块；
* 不允许因为发现新想法就偏离当前任务；
* 每轮修复最多允许进行固定次数，例如 3 次；
* 超过修复次数后，必须生成问题报告，而不是继续盲目修复；
* 只有影响运行、测试、需求一致性或安全性的缺陷才允许进入修复流程；
* 非关键优化必须进入 backlog，不能打断当前主线任务。

## 上下文管理要求

由于 AI Agent 可能发生上下文溢出、会话中断或工具切换，因此文档必须设计一套**不依赖聊天上下文**的项目记忆机制。

请要求 AI Agent 在项目中持续维护以下文件：

```text
/PROJECT_BRIEF.md          项目总说明
/REQUIREMENTS.md           需求文档
/ARCHITECTURE.md           架构设计
/ROADMAP.md                阶段路线图
/TASKS.md                  总任务清单
/TIME_SLICES.md            时间切片任务表
/STATUS.md                 当前进度状态
/DECISIONS.md              关键决策记录
/CHANGELOG.md              变更记录
/ISSUES.md                 问题与阻塞记录
/ACCEPTANCE.md             验收标准
/AGENT_HANDOFF.md          Agent 交接说明
```

每次执行任务前，AI Agent 必须先读取这些文件，而不是依赖历史聊天内容。

每次完成任务后，AI Agent 必须更新这些文件，以便下一个 Agent、下一轮会话或另一个工具可以无缝接手。
